CASO DE USO: MostrarClientes

IDENTIFICADOR: CU15

OBJETIVO: El administador con rol de gestión de usuarios desea obtener los datos de los clientes registrados en el sistema.

CONTEXTO:

1. El administrador debe estar registrado en el sistema.
2. El administrador debe tener rol de gestión de usuarios.

ACTOR PRINCIPAL: Administrador con rol de gestión de usuarios.

ESCENARIO PRINCIPAL

1. El administrador accede al sistema mediante su dni y su contraseña.
2. El sistema muestra un menu de tipos de funcionalidades (usuarios,máquinas,reservas)
3. El administrador accede al menú de funcionalidades de usuario.
4. El sistema muestra un menú de funcionalidades de usuario.
5. El administrador accede a la funcionalidad de mostrar clientes.
6. El sistema muestra una lista de los clientes registrados en el sistema.

EXTENSIONES

2a) El sistema notifica error si el administrador no está registrado en el sistema.
4a) El sistema notifica error si el administrador no tiene rol de gestión de usuarios.
